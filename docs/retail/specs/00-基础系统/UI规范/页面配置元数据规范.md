# 页面配置元数据规范

## 目的
统一前后端对“页面结构、字段、规则、交互”的描述方式，减少硬编码，提升可配置与可演进能力。

## 适用范围
适用于零售系统后台管理端的页面、表单、列表、详情页、弹窗、流程节点等 UI 场景。

## 基本原则
- 后端提供结构与规则，前端负责渲染与交互
- 元数据必须可版本化与可追溯
- 字段级权限与数据字典统一通过元数据表达
- 规则表达要可读、可验证、可回放

## 版本与兼容
- 元数据必须包含 `metaVersion` 与 `view.schemaVersion`
- 后端升级字段时必须保证向后兼容或提供迁移策略
- 前端只渲染已声明支持的 `view.schemaVersion`

## 数据格式约定
- 传输格式为 JSON
- 所有 ID 使用字符串
- 时间使用 ISO-8601 且标明时区偏移
- 金额字段必须声明币种与精度

## 国际化与本地化
- 元数据返回应为当前语言的直接文案（如 `view.name`、`field.label` 已本地化）
- 切换语言需要重新请求元数据（通过 `lang` 或 `Accept-Language`）
- 货币字段需声明 `currencyCode` 与 `precision`
- 时间展示需声明 `timeZone` 与 `format`

## 页面加载流程（菜单 -> 元数据 -> 业务数据）
1. 菜单/路由仅绑定页面元数据接口：`GET /view-meta/{viewKey}`
2. 元数据返回页面结构及 `dataSources` 列表（业务数据接口）
3. 前端根据 `dataSources` 发起业务数据请求并渲染页面
4. 切换语言或版本时重新请求元数据

## 命名规范
- 统一使用 lowerCamelCase
- 布尔类型使用 `is` 或 `has` 前缀
- 列表型字段用复数

## 顶层结构
顶层包含 `metaVersion`、`view`。视图版本通过 `view.schemaVersion` 标识。视图内容通过 `view.sections` 和 `view.rules` 体现层级关系。

**顶层示例**
```json
{
  "metaVersion": "1.0",
  "view": {
    "id": "product-edit",
    "schemaVersion": "2026.02",
    "sections": [],
    "rules": []
  }
}
```

## view 模型
定义视图自身信息与数据源。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 页面唯一 ID |
| schemaVersion | string | 是 | 页面结构版本 |
| name | string | 是 | 页面名称 |
| route | string | 是 | 前端路由路径 |
| layout | object | 是 | 页面布局定义 |
| permissions | object | 否 | 页面级权限 |
| dataSources | array | 否 | 页面数据源列表 |
| i18n | object | 否 | 语言/区域标识（可选，不承担翻译） |
| sections | array | 否 | 页面区块列表 |
| rules | array | 否 | 页面级规则 |

**示例**
```json
{
  "id": "product-edit",
  "name": "商品编辑",
  "route": "/mdm/product/edit",
  "layout": { "type": "form", "grid": 2, "responsive": true },
  "permissions": { "allow": ["mdm.product.edit"] },
  "dataSources": [
    {
      "id": "productDetail",
      "method": "GET",
      "url": "/api/mdm/products/{id}",
      "params": { "id": "route.id" },
      "mapping": { "id": "id", "name": "name", "categoryId": "categoryId" }
    }
  ],
  "i18n": { "lang": "zh-CN" },
  "sections": [],
  "rules": []
}
```

## layout 模型
描述页面整体布局。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| type | string | 是 | 布局类型，例 `form`、`list`、`detail` |
| grid | number | 否 | 栅格列数 |
| responsive | boolean | 否 | 是否启用响应式 |

**示例**
```json
{ "type": "form", "grid": 2, "responsive": true }
```

## layout 类型与协同关系

### 布局类型（layout.type）建议枚举
| 类型 | 说明 | 典型场景 |
| --- | --- | --- |
| form | 表单编辑型 | 新增/编辑页面 |
| list | 检索列表型 | 列表查询页 |
| detail | 详情展示型 | 详情页 |
| split | 左右分栏型 | 左树右表/左表右详情 |
| tabs | 页签型 | 复杂对象分区编辑 |
| wizard | 向导型 | 多步骤流程 |

### 协同关系（layout 与 sections/components）
- `layout` 决定页面骨架与主结构。
- `sections` 是 layout 下的逻辑区块（如“基础信息”“查询条件”“步骤一”）。
- `components` 是区块内可渲染组件（如 form、table、card、tree）。

### 典型组合与示例

**1) form**
- sections: 基础信息、扩展信息
- components: form、card、table（子表）

```json
{
  "layout": { "type": "form", "grid": 2 },
  "sections": [
    { "id": "basic", "title": "基础信息", "components": [{ "id": "basicForm", "type": "form" }] },
    { "id": "ext", "title": "扩展信息", "components": [{ "id": "extCard", "type": "card" }] }
  ]
}
```

**页面示意（form）**
```text
┌──────────────────────────────────────────────┐
│ 标题区：商品编辑                              │
├──────────────────────────────────────────────┤
│ 基础信息（form）                              │
│ [名称][品类]                                  │
│ [规格][单位]                                  │
├──────────────────────────────────────────────┤
│ 扩展信息（card/table）                        │
│ 子表：条码/属性/图片                          │
└──────────────────────────────────────────────┘
```


**2) list**
- sections: 查询条件、列表
- components: filterForm、table、pagination

```json
{
  "layout": { "type": "list" },
  "sections": [
    { "id": "filters", "title": "查询条件", "components": [{ "id": "filterForm", "type": "form" }] },
    { "id": "result", "title": "结果列表", "components": [{ "id": "resultTable", "type": "table" }] }
  ]
}
```

**页面示意（list）**
```text
┌──────────────────────────────────────────────┐
│ 标题区：商品列表                              │
├──────────────────────────────────────────────┤
│ 查询条件（form）                              │
│ [名称][品类][状态]   [查询][重置]             │
├──────────────────────────────────────────────┤
│ 列表（table）                                 │
│ 名称 | 品类 | 状态 | 更新时间                 │
└──────────────────────────────────────────────┘
```


**3) detail**
- sections: 基本信息、关联信息、操作记录
- components: descriptions、table、timeline

```json
{
  "layout": { "type": "detail" },
  "sections": [
    { "id": "info", "title": "基本信息", "components": [{ "id": "infoCard", "type": "descriptions" }] },
    { "id": "logs", "title": "操作记录", "components": [{ "id": "opTimeline", "type": "timeline" }] }
  ]
}
```

**页面示意（detail）**
```text
┌──────────────────────────────────────────────┐
│ 标题区：商品详情  [在售]  编码:P0001          │
├──────────────────────────────────────────────┤
│ 基本信息（descriptions/detailCard）           │
│ 名称 | 品类 | 品牌 | 规格 | 单位 | 创建时间   │
├──────────────────────────────────────────────┤
│ 关联信息（table）                             │
│ 供应商 | 供货价 | 起始日期 | 状态            │
├──────────────────────────────────────────────┤
│ 操作记录（timeline）                          │
│ 2026-02-20 创建 / 2026-02-25 变更             │
└──────────────────────────────────────────────┘
```


**4) split**
- sections: left、right
- components: left=tree/table，right=detail/form

```json
{
  "layout": { "type": "split" },
  "sections": [
    { "id": "left", "components": [{ "id": "leftTree", "type": "tree" }] },
    { "id": "right", "components": [{ "id": "rightDetail", "type": "detail" }] }
  ]
}
```

**页面示意（split）**
```text
┌──────────────────────────────────────────────┐
│ 左侧（tree/table）   │ 右侧（detail/form）    │
│ 组织树/品类树         │ 详情或编辑表单         │
└──────────────────────────────────────────────┘
```


**5) tabs**
- sections: 每个 section 对应一个 tab
- components: tab 内可放 form/table/card

```json
{
  "layout": { "type": "tabs" },
  "sections": [
    { "id": "tab1", "title": "基本", "components": [{ "id": "baseForm", "type": "form" }] },
    { "id": "tab2", "title": "价格", "components": [{ "id": "priceTable", "type": "table" }] }
  ]
}
```

**页面示意（tabs）**
```text
┌──────────────────────────────────────────────┐
│ Tab: 基本 | 价格 | 供应商 | 库存             │
├──────────────────────────────────────────────┤
│ 当前 Tab 内容（form/table/card）             │
└──────────────────────────────────────────────┘
```


**6) wizard**
- sections: 每个 section 对应一步
- components: step 内多为 form + actions

```json
{
  "layout": { "type": "wizard" },
  "sections": [
    { "id": "step1", "title": "步骤一", "components": [{ "id": "step1Form", "type": "form" }] },
    { "id": "step2", "title": "步骤二", "components": [{ "id": "step2Form", "type": "form" }] }

```

```text
│ 当前步骤表单（form）                          │
│ [上一页] [下一页]                             │
└──────────────────────────────────────────────┘
```



## section 模型
页面的逻辑区块。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 区块唯一 ID |
| title | string | 否 | 区块标题 |
| description | string | 否 | 区块说明 |
| components | array | 是 | 组件列表 |

**示例**
```json
{
  "id": "basic",
  "title": "基础信息",
  "components": []
}
```


**与 layout 的协同**
- `section` 是 layout 下的逻辑区块，数量与含义由 `layout.type` 决定。
- `tabs` 或 `wizard` 布局下，`section` 直接对应页签或步骤。
- `split` 布局下，`section` 通常固定为 `left` 与 `right`。


## component 模型
定义具体 UI 组件与其字段。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 组件唯一 ID |
| type | string | 是 | 组件类型，例 `form`、`table`、`card`、`tabs` |
| props | object | 否 | 组件属性 |
| fields | array | 否 | 字段列表 |
| actions | array | 否 | 操作按钮 |
| rules | array | 否 | 组件级规则 |

**示例**
```json
{
  "id": "basicForm",
  "type": "form",
  "props": { "labelWidth": 120 },
  "fields": [],
  "actions": []
}
```


**与 section/layout 的协同**
- `component` 必须挂在 `section` 内，决定可渲染内容。
- `layout.type` 会约束可用的 `component.type` 组合（如 `list` 通常由查询表单 + 表格组成）。
- 同一 `section` 内组件按声明顺序渲染。


## field 模型
页面可配置字段的核心定义，`type` 表达字段的显示方案与交互类型，数据类型可通过 `format` 或校验规则体现。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 字段唯一 ID |
| label | string | 否 | 字段显示名（已本地化） |
| type | string | 是 | 字段类型，例 `input`、`select`、`treeSelect`、`date` |
| required | boolean | 否 | 是否必填 |
| readOnly | boolean | 否 | 是否只读 |
| visible | boolean | 否 | 是否可见 |
| defaultValue | any | 否 | 默认值 |
| options | object | 否 | 选项定义 |
| validators | array | 否 | 校验规则列表 |
| i18n | object | 否 | 语言/区域标识（可选，不承担翻译） |
| permissions | object | 否 | 字段权限 |
| binding | string | 否 | 数据绑定路径 |
| format | string | 否 | 展示格式，例日期、金额 |

**示例**
```json
{
  "id": "categoryId",
  "label": "品类",
  "type": "treeSelect",
  "required": true,
  "options": { "type": "api", "source": "/api/mdm/categories" },
  "binding": "productDetail.categoryId"
}
```

## action 模型
页面或组件上的操作定义。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 操作唯一 ID |
| label | string | 是 | 操作名称 |
| type | string | 是 | 操作类型，例 `submit`、`navigate`、`dialog` |
| api | object | 否 | 调用 API 定义 |
| confirm | object | 否 | 确认提示 |
| successMessage | string | 否 | 成功提示 |
| errorMessage | string | 否 | 失败提示 |
| permissions | object | 否 | 操作权限 |

**示例**
```json
{
  "id": "save",
  "label": "保存",
  "type": "submit",
  "api": { "method": "POST", "url": "/api/mdm/products" },
  "successMessage": "保存成功"
}
```

**提交载荷约定（默认）**
- 提交时按 `dataSource` 包裹结构提交
- 字段以 `binding` 所属的 `dataSourceId` 作为顶层对象名

**示例（dataSource 包裹）**
```json
{
  "productDetail": {
    "name": "测试商品",
    "categoryId": "1001"
  }
}
```

## rules 模型
用于表达条件与行为。
规则可挂在 `view.rules`（视图级）或 `component.rules`（组件级），就近控制作用范围。
同一目标冲突时，`component.rules` 优先于 `view.rules`。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| when | object | 是 | 条件表达式 |
| then | object | 是 | 行为表达式 |
| scope | string | 否 | 作用范围，例 `view`、`component` |

**规则语法（JSON Logic）**
- `when` 使用 JSON Logic 表达式
- 字段取值通过 `{"var": "fieldId"}` 获取
- 复杂条件通过 `and`/`or` 组合

**常用 JSON Logic 操作符**
- `==`
- `!=`
- `>`
- `<`
- `>=`
- `<=`
- `in`
- `and`
- `or`
- `!`

**行为操作**
- `setVisible`
- `setReadOnly`
- `setRequired`
- `setValue`
- `setOptions`

**示例**
```json
{
  "when": { "==": [ { "var": "categoryId" }, "FOOD" ] },
  "then": { "setRequired": ["shelfLife"] },
  "scope": "view"
}
```

**JSON Logic 示例（组合条件）**
```json
{
  "when": {
    "and": [
      { "==": [ { "var": "categoryId" }, "FRESH" ] },
      { ">": [ { "var": "shelfLife" }, 0 ] }
    ]
  },
  "then": { "setRequired": ["coldChainRequirement"] }
}
```

## dataSource 模型
页面数据源定义。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| id | string | 是 | 数据源 ID |
| method | string | 是 | HTTP 方法 |
| url | string | 是 | API 地址 |
| params | object | 否 | 入参映射 |
| mapping | object | 否 | 响应字段映射 |
| cache | object | 否 | 缓存策略 |

**示例**
```json
{
  "id": "productDetail",
  "method": "GET",
  "url": "/api/mdm/products/{id}",
  "params": { "id": "route.id" },
  "mapping": { "id": "id", "name": "name" }
}
```

## options 模型
用于 select、radio、checkbox 等选项定义。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| type | string | 是 | `static` 或 `api` |
| items | array | 否 | 静态选项 |
| source | string | 否 | 选项接口 |
| labelKey | string | 否 | 选项显示字段 |
| valueKey | string | 否 | 选项值字段 |

**示例**
```json
{ "type": "static", "items": [{ "label": "启用", "value": "Y" }] }
```

## validators 模型
字段校验规则定义。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| type | string | 是 | 校验类型，例 `required`、`length`、`regex` |
| message | string | 否 | 错误提示 |
| params | object | 否 | 规则参数 |

**示例**
```json
{ "type": "length", "params": { "min": 2, "max": 50 } }
```

**校验执行约束**
- 前端用于交互校验提示，后端必须执行同口径强校验
- 同一字段多个校验按声明顺序执行，失败即返回首个错误
- 跨字段校验可通过 `type: "jsonLogic"` 表达，条件写在 `params.expr`

**跨字段校验示例**
```json
{
  "type": "jsonLogic",
  "message": "基础进价必须小于基础售价",
  "params": {
    "expr": {
      "<": [
        { "var": "productDetail.basePurchasePrice" },
        { "var": "productDetail.baseSalePrice" }
      ]
    }
  }
}
```

## permissions 模型
权限表达统一结构。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| allow | array | 否 | 允许权限点 |
| deny | array | 否 | 禁止权限点 |
| orgScope | array | 否 | 组织范围限制 |

**示例**
```json
{ "allow": ["mdm.product.edit"], "orgScope": ["HQ", "REGION"] }
```

## i18n 模型
语言/区域标识定义（可选）。

**字段说明**
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| lang | string | 否 | 语言标识，如 `zh-CN` |
| region | string | 否 | 区域标识，如 `CN` |
| currencyCode | string | 否 | 货币代码 |
| precision | number | 否 | 金额精度 |
| timeZone | string | 否 | 时区 |
| format | string | 否 | 时间或数字格式 |

**示例**
```json
{ "lang": "zh-CN", "currencyCode": "CNY", "precision": 2 }
```

## binding 规则
`binding` 表达字段与数据源的绑定路径，格式为 `dataSourceId.fieldPath`。

**示例**
```json
{ "binding": "productDetail.name" }
```

## 视图模型与领域模型的映射
当领域模型仅包含 ID 而页面需要显示名称/描述时，应在领域模型之上构建视图模型（View Model）。页面字段应绑定到视图模型字段，而不是直接绑定领域模型字段。

**示例：领域模型与视图模型**
```json
{
  "domain": {
    "product": { "id": "P001", "categoryId": "1001" }
  },
  "viewModel": {
    "productDetail": { "id": "P001", "categoryId": "1001", "categoryName": "生鲜" }
  }
}
```

**示例：页面绑定视图模型字段**
```json
{
  "id": "categoryName",
  "type": "text",
  "binding": "productDetail.categoryName"
}
```

**视图模型构建来源**
- 字典/主数据映射（如 `categoryId -> categoryName`）
- 关联对象展开（如 `supplierId -> supplierName`）
- 聚合查询结果（如价格、库存等汇总字段）

**dataSource 与 viewModel 对应示例**
- `productDetail` -> `viewModel.productDetail`
- `categoryOptions` -> `viewModel.categoryOptions`

**运行时 viewModel 示例**
```json
{
  "viewModel": {
    "productDetail": {
      "id": "P001",
      "name": "冷鲜牛奶",
      "categoryId": "1001",
      "categoryName": "生鲜"
    },
    "categoryOptions": [
      { "id": "1001", "name": "生鲜" },
      { "id": "1002", "name": "日配" }
    ]
  }
}
```

## 完整示例
```json
{
  "metaVersion": "1.0",
  "view": {
    "id": "product-edit",
    "schemaVersion": "2026.02",
```
