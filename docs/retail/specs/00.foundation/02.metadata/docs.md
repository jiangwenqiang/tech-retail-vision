# SaaS 多租户元数据设计

本目录以 AtroCore 元数据模型为参考，目标是构建“平台基线元数据 + 租户自定义”的统一治理体系。

## 目录结构

1. [01. 总览与边界](./01.overview/docs)
2. [02. 平台基线元数据模型](./02.baseline-metadata/docs)
3. [03. 租户元数据扩展模型](./03.tenant-metadata/docs)
4. [04. 运行时与存储模型](./04.runtime-storage/docs)
5. [05. 生命周期与治理](./05.governance/docs)

## 设计主线

1. 基线统一定义：平台维护标准实体、字段类型、字段、属性、客户端模型
2. 租户按范围扩展：通过可见范围、覆盖规则、提升机制支持租户差异化
3. 发布受控：所有元数据变更受版本、门禁、回滚机制约束
