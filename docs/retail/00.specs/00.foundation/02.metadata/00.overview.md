# SaaS 元数据管理系统

基于元数据模型的完整实现，支持平台基线和租户级别的元数据管理。每个对象采用最小化实现，预留扩展空间。

## 基线与租户管理机制

### 作用域规则

| 作用域   | 定义者     | 可见性   | 可修改性     |
| -------- | ---------- | -------- | ------------ |
| PLATFORM | 平台管理员 | 所有租户 | 仅平台管理员 |
| TENANT   | 租户管理员 | 当前租户 | 租户管理员   |

### Customization - 定制配置

Customization 是一个统一的配置对象，用于管理租户对平台基线元数据的定制化配置。详细配置说明见：[06.customization.md](./06.customization.md)

**核心设计**：
- **通用结构**：通过 `targetType` + `targetCode` + `properties` 的统一结构支持所有元数据对象
- **状态一致**：通过 `status` 字段统一控制元数据对象的启用/禁用状态
- **可扩展**：易于添加新的目标类型和可覆盖属性
- **权限控制**：只能定制 `customizable=true` 的元数据对象

**支持的目标类型**：
- field（字段）- 可覆盖 name、required、readOnly、status
- attribute（属性）- 可覆盖 name、required、status
- client（客户端配置）- 可覆盖完整配置
- layout（布局配置）- 可覆盖完整配置

### 可见性管理

| 元数据对象 | 平台级可见性控制 | 租户级可见性控制 | 说明 |
|-----------|----------------|----------------|------|
| Entity Definition | status + scope | - | 只有 status=ACTIVE 的 PLATFORM 实体才对所有租户可见 |
| Field | status | Customization（覆盖 status） | 通过 Customization 覆盖 status 为 DISABLED 来禁用可定制字段 |
| Attribute | status | Customization（覆盖 status） | 通过 Customization 覆盖 status 为 DISABLED 来禁用可定制属性 |
| Client Definition | - | - | 通过 scope=TENANT 覆盖 |
| Layout | - | - | 通过 scope=TENANT 覆盖 |

**可见性规则**：
- **实体可见性**：通过 `status` 和 `scope` 字段控制，只有 `status=ACTIVE` 的 PLATFORM 实体才对所有租户可见
- **字段/属性可见性**：通过 `status` 字段控制，租户可通过 Customization 覆盖 status 为 DISABLED 来禁用可定制元数据
- **状态一致性**：所有元数据对象使用统一的状态管理机制（ACTIVE/DISABLED/DRAFT）
- **配置优先级**：租户覆盖的 status > 平台默认 status
- **运行时合并**：先检查实体的 status，再应用租户 Customization 配置，只返回 status=ACTIVE 的元数据对象

### 定制化管理

| 元数据对象 | 平台级控制 | 租户级定制 | 说明 |
|-----------|-----------|-----------|------|
| Entity | customizable 字段 | - | 控制是否允许租户定制该实体 |
| Field | customizable 字段 | Customization（属性覆盖） | 只能定制标记为可定制的字段 |
| Attribute | customizable 字段 | Customization（属性覆盖） | 只能定制标记为可定制的属性 |
| Attribute（新增） | - | scope=TENANT | 租户可添加自定义属性 |
| Client Definition | customizable 字段 | Customization（属性覆盖） | 只能定制标记为可定制的客户端配置 |
| Layout | customizable 字段 | Customization（属性覆盖） | 只能定制标记为可定制的布局配置 |

**定制化规则**：
- 所有元数据对象都有 `customizable` 字段，控制是否允许租户定制
- 只有 `customizable=true` 的元数据对象才能被租户定制
- 通过 Customization 的统一结构进行定制：`targetType` + `targetCode` + `properties`
- 租户定制优先于平台基线
- 运行时合并：租户配置 > 平台默认

**Customization 支持的属性覆盖**：
- **Field**：name、required、readOnly、status
- **Attribute**：name、required、status
- **Client/Layout**：完整配置对象

### 覆盖规则

**允许覆盖**（需 `customizable=true`）：

- 字段的 name、required、readOnly、status
- 属性的 name、required、status
- 客户端配置（Client、Layout）

**禁止覆盖**：

- code、type、scope 等关键字段
- fields、links、indexes 等结构定义
- `customizable=false` 的元数据对象

**状态覆盖优先级**：
- 租户覆盖的 status > 平台默认 status
- status=DISABLED 的对象不会在运行时返回
- 通过覆盖 status 字段实现字段/属性的禁用功能

### 租户扩展能力

**1. 字段可见性控制**：

- 可以禁用平台定义的字段（如果字段的 `customizable=true`）
- 不能新增字段
- 通过 Customization 覆盖 status 为 DISABLED 实现：
  ```json
  {
    "targetType": "field",
    "targetCode": "internalNotes",
    "properties": {
      "status": "DISABLED"
    }
  }
  ```

**2. 属性扩展**：

- 可以添加租户级别的 Attribute（`scope=TENANT`）
- 可以禁用平台基线的 Attribute（如果属性的 `customizable=true`）
- 属性与字段类似，支持动态扩展
- 通过 Customization 覆盖 status 为 DISABLED 实现禁用

**3. 属性覆盖**：

- 可以覆盖字段和属性的配置（仅限 `customizable=true` 的元数据）
- 可以覆盖客户端配置（Client、Layout）（如果实体的 `customizable=true`）
- 通过 Customization 覆盖属性实现：
  ```json
  {
    "targetType": "field",
    "targetCode": "productName",
    "properties": {
      "name": "商品名称",
      "required": false
    }
  }
  ```

**注意**：所有元数据对象使用统一的状态管理机制，通过 `status` 字段控制启用/禁用状态，确保设计的一致性。

## 元数据合并机制

元数据合并将平台基线元数据与租户定制配置合并，生成租户专属的元数据视图。详细的技术实现方案见：[../../01.architecture/00.foundation/02.metadata/](../../01.architecture/00.foundation/02.metadata/)

### 合并策略

- **属性覆盖**：通过 Customization 配置覆盖平台基线元数据的属性
- **状态过滤**：过滤 `status=DISABLED` 的元数据对象
- **权限控制**：只合并 `customizable=true` 的对象
- **配置优先级**：租户配置 > 平台默认

### 合并流程

1. 实体级别：检查实体的 `status` 和 `customizable` 属性
2. 字段级别：合并字段属性，过滤禁用字段
3. 属性级别：合并属性配置，添加租户自定义属性
4. 配置级别：合并客户端配置和布局配置

## 数据模型关系

```
Entity Definition (1) ──< (N) Field
Entity Definition (1) ──< (N) Link
Entity Definition (1) ──< (N) Index
Entity Definition (1) ──< (N) Attribute
Entity Definition (1) ──< (N) Client
Entity Definition (1) ──< (N) Layout
Attribute (1) ──< (N) Attribute (自关联)
Customization (N) ──< (1) Entity Definition (通过 entityId 关联)
Customization 根据 targetType 定制各种元数据对象
```
