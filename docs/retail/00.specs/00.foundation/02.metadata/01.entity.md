# Entity Definition - 实体定义

## 作用

Entity Definition 定义实体的完整结构，包括实体能力属性、字段、关系和索引。

## 结构组成

```
Entity Definition
├── props (基础属性)
├── fields（字段定义）
├── links（关系定义）
└── indexes（索引定义）
```

## props（基础属性）

### 核心字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| code | VARCHAR(100) | 是 | - | 实体编码（全局唯一，创建后不可修改） |
| name | VARCHAR(255) | 是 | - | 实体名称 |
| namePlural | VARCHAR(255) | 否 | - | 实体名称（复数） |
| type | VARCHAR(50) | 是 | Base | 实体类型 |
| scope | VARCHAR(20) | 是 | PLATFORM | 作用域（PLATFORM/TENANT） |
| customizable | BOOLEAN | 是 | true | 是否允许租户定制 |
| status | VARCHAR(20) | 是 | DRAFT | 状态（DRAFT/ACTIVE/DISABLED） |
| iconClass | VARCHAR(100) | 否 | - | 图标样式类 |
| color | VARCHAR(30) | 否 | - | 显示颜色 |
| version | INT | 是 | 1 | 当前版本号 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

### 实体类型（type）

| 类型 | 代码 | 说明 | 使用场景 |
|-----|------|------|---------|
| 基础实体 | Base | 标准业务对象 | 产品、客户、订单 |
| 层级实体 | Hierarchy | 支持父子关系 | 分类树、组织架构 |
| 关系实体 | Relation | 多对多中间表 | 产品-渠道、客户-标签 |
| 参考数据 | ReferenceData | 枚举字典 | 国家、币种、单位 |
| 归档实体 | Archive | 历史日志 | 操作日志、审计记录 |

### 状态（status）

| 状态 | 说明 |
|-----|------|
| DRAFT | 草稿，未生效 |
| ACTIVE | 生效中 |
| DISABLED | 已禁用 |

### 实体能力配置示例

```json
{
  "code": "Product",
  "name": "产品",
  "namePlural": "产品",
  "type": "Base",
  "scope": "PLATFORM",
  "customizable": true,
  "status": "ACTIVE"
}
```

## fields（字段定义）

### 核心字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| entityCode | VARCHAR(100) | 是 | - | 关联实体编码（Entity.code） |
| code | VARCHAR(100) | 是 | - | 字段编码（实体内唯一） |
| name | VARCHAR(255) | 是 | - | 字段名称 |
| type | VARCHAR(50) | 是 | - | 字段类型 |
| scope | VARCHAR(20) | 是 | PLATFORM | 作用域（PLATFORM/TENANT） |
| customizable | BOOLEAN | 是 | true | 是否允许租户定制 |
| required | BOOLEAN | 是 | false | 是否必填 |
| readOnly | BOOLEAN | 是 | false | 是否只读 |
| status | VARCHAR(20) | 是 | ACTIVE | 状态（DRAFT/ACTIVE/DISABLED） |
| defaultValue | VARCHAR(500) | 否 | - | 默认值 |
| params | JSON | 否 | - | 参数配置 |
| sortOrder | INT | 是 | - | 排序号 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

### 字段示例

```json
{
  "entityCode": "Product",
  "code": "productName",
  "name": "产品名称",
  "type": "varchar",
  "scope": "PLATFORM",
  "required": true,
  "readOnly": false,
  "status": "ACTIVE",
  "params": {
    "maxLength": 255
  },
  "sortOrder": 1
}
```

## links（关系定义）

### 核心字段

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| entityCode | VARCHAR(100) | 是 | 关联实体编码（Entity.code） |
| code | VARCHAR(100) | 是 | 关系编码（实体内唯一） |
| name | VARCHAR(255) | 是 | 关系名称 |
| type | VARCHAR(50) | 是 | 关系类型 |
| foreignEntity | VARCHAR(100) | 是 | 关联实体 |
| foreign | VARCHAR(100) | 否 | 反向关系字段 |
| required | BOOLEAN | 是 | 是否必填 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

### 关系类型

| 类型 | 代码 | 说明 | 示例 |
|-----|------|------|------|
| 一对一 | oneToOne | 单值关联 | 用户 - 档案 |
| 一对多 | oneToMany | 单向一对多 | 订单 - 订单项 |
| 多对一 | manyToOne | 反向一对多 | 订单项 - 订单 |
| 多对多 | manyToMany | 多值关联 | 产品 - 分类 |

### 关系示例

```json
{
  "entityCode": "Product",
  "code": "category",
  "name": "分类",
  "type": "manyToOne",
  "foreignEntity": "Category",
  "foreign": "products",
  "required": true
}
```

## indexes（索引定义）

### 核心字段

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| entityCode | VARCHAR(100) | 是 | 关联实体编码（Entity.code） |
| code | VARCHAR(100) | 是 | 索引编码 |
| fields | ARRAY | 是 | 字段列表 |
| unique | BOOLEAN | 是 | 是否唯一索引 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

### 索引示例

```json
{
  "entityCode": "Product",
  "code": "idx_product_code",
  "fields": ["productCode"],
  "unique": true
}
```

## 基线与租户管理

### 平台基线实体（PLATFORM）

- 平台管理员创建和修改
- 根据 `status` 字段决定是否生效（ACTIVE 才可见）
- 根据 `scope` 字段决定作用域
- 租户不能删除或修改定义
- 当 `customizable=true` 时，租户可通过 Metadata Customization 配置来：
  - 覆盖字段和属性（包括通过 `status=DISABLED` 实现租户级禁用）

### 租户实体配置（TENANT）

- 租户不能创建独立的实体
- 只能通过 Metadata Customization 配置定制化平台实体
- 只能添加或通过 `status` 禁用租户级别的 Attribute

### 可见性控制

**实体级别**：
- 通过 `status` 字段控制：只有 `status=ACTIVE` 的实体才可见
- 通过 `scope` 字段控制：PLATFORM 作用域的实体对所有租户可见（如果 status=ACTIVE）

**字段/属性级别**（通过 Metadata Customization）：
- 租户可以覆盖平台定义字段/属性的 `status`
- 当 `status=DISABLED` 时对象在该租户下不可见（前提：目标对象 `customizable=true`）

## 约束规则

1. **实体归属约束**：`Field`、`Link`、`Index` 必须显式包含 `entityCode`，且 `entityCode` 必须引用有效 `Entity.code`
2. **字段唯一性**：`Field.code + entityCode + yn` 必须唯一
3. **关系唯一性**：`Link.code + entityCode + yn` 必须唯一
4. **索引唯一性**：`Index.code + entityCode + yn` 必须唯一
5. **索引字段归属**：`Index.fields[*]` 中每个字段必须存在于同一 `entityCode` 下的 Field 集合
6. **类型限制**：`Field.type` 必须在字段类型白名单中
7. **参数契约**：`Field.params` 必须符合字段类型的参数白名单
8. **关联验证**：`Link.foreignEntity` 必须存在
9. **可见性控制**：
   - 平台元数据通过 `status` 控制全局可见性（`ACTIVE` 才可见）
   - 租户通过 Metadata Customization 覆盖对象 `status` 控制租户级可见性（`DISABLED` 不可见）
10. **定制化控制**：只有当 `customizable=true` 时，租户才能定制该元数据对象
11. **实体编码不可变**：`Entity.code` 创建后禁止修改；如需调整编码，必须走“新建实体编码 + 元数据迁移 + 旧编码下线”流程
