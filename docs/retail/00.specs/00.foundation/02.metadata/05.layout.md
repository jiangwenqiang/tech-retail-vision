# Layout Definition - 布局编排定义

## 作用

Layout 定义页面布局和面板编排。

## 核心字段

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| scope | VARCHAR(20) | 是 | 作用域（PLATFORM/TENANT） |
| type | VARCHAR(50) | 是 | 布局类型 |
| customizable | BOOLEAN | 是 | true | 是否允许租户自定义 |
| layout | JSON | 是 | 布局配置 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

## 布局类型

| 类型 | 代码 | 说明 |
|-----|------|------|
| 列表布局 | list | 列表页配置 |
| 详情布局 | detail | 详情页配置 |
| 表单布局 | form | 表单配置 |
| 关系布局 | relationship | 关系面板配置 |

## 列表布局示例

```json
{
  "scope": "PLATFORM",
  "type": "list",
  "layout": {
    "columns": [
      {"field": "name", "width": 200, "sortable": true},
      {"field": "code", "width": 150, "sortable": true},
      {"field": "status", "width": 100}
    ],
    "filters": [
      {"field": "status", "operator": "equals"}
    ],
    "actions": ["create", "edit", "delete"]
  }
}
```

## 详情布局示例

```json
{
  "scope": "PLATFORM",
  "type": "detail",
  "layout": {
    "panels": [
      {
        "name": "基本信息",
        "rows": [
          [
            {"field": "name"},
            {"field": "code"}
          ],
          [
            {"field": "status"},
            {"field": "createdAt"}
          ]
        ]
      }
    ]
  }
}
```

## 基线与租户管理

### 平台布局（PLATFORM）
- 平台定义默认布局
- 所有租户可见
- 租户不能删除或修改平台布局

### 租户布局（TENANT）
- 当 `customizable = true` 时，租户可以完全覆盖布局
- 租户布局仅当前租户可见
- 运行时：租户布局优先于平台布局

### 可见性与定制化
- Layout Definition 本身不支持可见性控制
- 通过实体的 `customizable` 字段控制是否允许租户定制布局配置
