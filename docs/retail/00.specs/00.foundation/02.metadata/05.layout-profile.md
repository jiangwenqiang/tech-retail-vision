# Layout Profile - 布局方案定义

## 作用

Layout Profile 定义一组可切换的布局方案模板，不直接绑定实体；实体通过 Layout 选择并绑定具体 Profile。

## 核心字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| code | VARCHAR(100) | 是 | default | 方案编码（同一 `scope+owner` 内唯一） |
| name | VARCHAR(255) | 是 | - | 方案名称 |
| scope | VARCHAR(20) | 是 | PLATFORM | 作用域（PLATFORM/TENANT/USER） |
| tenantId | BIGINT | 否 | - | 租户ID（`scope=PLATFORM` 为空；`scope=TENANT/USER` 必填） |
| owner | VARCHAR(100) | 是 | platform | 归属主体（按 scope 解释） |
| status | VARCHAR(20) | 是 | DRAFT | 状态（DRAFT/ACTIVE/DISABLED） |
| customizable | BOOLEAN | 是 | true | 是否允许下一层 scope 派生该方案 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

### 字段语义与约束

- `tenantId`：租户归属键。`scope=PLATFORM` 时为空；`scope=TENANT/USER` 时必填
- `owner`：归属主体。`scope=PLATFORM` 时固定 `platform`；`scope=TENANT` 时必须等于 `tenantId`；`scope=USER` 时为 `userId`
- `status`：方案发布状态，只有 `ACTIVE` 才能被运行时选用
- `customizable`：派生闸门，控制是否允许下一层 scope 基于该 Profile 派生新 Profile
- `scope=PLATFORM` 且 `customizable=true`：允许 Tenant 派生
- `scope=TENANT` 且 `customizable=true`：允许同租户 User 派生
- `scope=USER`：固定 `customizable=false`（终态，不再向下派生）
- 与 `Layout.customizable` 的关系：先判定 `LayoutProfile.customizable`（是否可派生），再判定 `Layout.customizable`（是否可修改条目）
- `scope=PLATFORM` 时 `tenantId` 必须为空，且 `owner=platform`
- `scope=TENANT` 时 `tenantId` 必填，且 `owner` 必须等于 `tenantId`
- `scope=USER` 时 `tenantId` 必填，`owner` 必须是有效 `userId`（`userId` 全局唯一）且该用户属于 `tenantId`
- `scope+owner+code` 必须唯一（业务唯一键，不包含 `yn`）
- 同一 `scope+owner` 下最多允许 1 条 `status=ACTIVE` 的 Profile
- `yn` 仅用于软删除标记，不参与业务唯一性判定

## 模型关系

- LayoutProfile (1) -> (N) Layout
- Layout 通过 `profileId + entityCode` 绑定具体实体下的布局条目

## 配置示例

```json
{
  "code": "default",
  "name": "标准布局方案",
  "scope": "PLATFORM",
  "tenantId": null,
  "owner": "platform",
  "status": "ACTIVE",
  "customizable": true
}
```

## 运行时选择建议

布局方案选择优先级建议：

1. 用户归属方案（`scope=USER, tenantId=tenantId, owner=userId`）
2. 租户归属方案（`scope=TENANT, tenantId=tenantId, owner=tenantId`）
3. 平台方案（`scope=PLATFORM, tenantId=null, owner=platform`）
4. 每层内优先使用显式指定的 `profile.code`；若未指定则使用该层 `status=ACTIVE` 的 Profile
5. 若当前层对目标实体未定义可用 Layout，则回退到下一层（`USER -> TENANT -> PLATFORM`）

## 与三模型定制的关系

- View 定制统一通过“派生 Profile”实现，不再区分微调或大改
- Tenant/User 的自定义本质是创建或更新其归属的 LayoutProfile；实体是否使用该 Profile 由 Layout 绑定决定
