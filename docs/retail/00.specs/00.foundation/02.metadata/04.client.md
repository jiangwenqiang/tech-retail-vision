# Client Definition - 客户端定义

## 作用

Client 定义前端展示和交互配置。

## 核心字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| entityCode | VARCHAR(100) | 是 | - | 关联实体编码（Entity.code） |
| scope | VARCHAR(20) | 是 | PLATFORM | 作用域（固定为 PLATFORM） |
| customizable | BOOLEAN | 是 | true | 是否允许租户自定义 |
| controller | VARCHAR(255) | 否 | - | 控制器路径 |
| views | JSON | 否 | - | 视图配置 |
| recordViews | JSON | 否 | - | 记录视图配置 |
| relationshipPanels | JSON | 否 | - | 关系面板配置 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

## 配置示例

```json
{
  "entityCode": "Product",
  "scope": "PLATFORM",
  "controller": "controllers/record",
  "views": {
    "list": "views/record/list",
    "detail": "views/record/detail",
    "edit": "views/record/edit"
  },
  "recordViews": {
    "list": "views/record/panels/list",
    "detail": "views/record/panels/detail"
  },
  "relationshipPanels": {
    "category": {
      "layout": "list",
      "select": true,
      "create": true
    }
  }
}
```

## 基线与租户管理

### 平台配置（PLATFORM）
- 平台定义默认配置
- 所有租户可见
- 租户不能直接修改平台配置
- 基线键为 `entityCode + scope + yn`，确保每个实体仅一套平台基线 Client

### 租户定制（Client Customization）
- 当 `customizable = true` 时，允许租户定义独立的 Tenant Client 配置
- Tenant Client 仅作用于当前租户，不影响平台基线
- 运行时：Tenant Client > Platform Client

### 可见性与定制化
- Client Definition 本身不支持可见性控制
- Client 不独立维护 `status`，其生效由关联实体状态与 Tenant/Platform Client 配置共同决定
- 通过实体与 Client Definition 的 `customizable` 字段共同控制是否允许租户定制客户端配置
- `entityCode` 必须指向有效 `Entity.code`，禁止“无实体归属”的 Client 配置
