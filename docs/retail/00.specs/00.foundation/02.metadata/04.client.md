# Client Definition - 客户端定义

## 作用

Client 定义前端展示和交互配置。

## 核心字段

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| scope | VARCHAR(20) | 是 | 作用域（PLATFORM/TENANT） |
| customizable | BOOLEAN | 是 | true | 是否允许租户自定义 |
| controller | VARCHAR(255) | 否 | 控制器路径 |
| views | JSON | 否 | 视图配置 |
| recordViews | JSON | 否 | 记录视图配置 |
| relationshipPanels | JSON | 否 | 关系面板配置 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

## 配置示例

```json
{
  "scope": "PLATFORM",
  "controller": "controllers/record",
  "views": {
    "list": "views/record/list",
    "detail": "views/record/detail",
    "edit": "views/record/edit"
  },
  "recordViews": {
    "list": "views/record/panels/list",
    "detail": "views/record/panels/detail"
  },
  "relationshipPanels": {
    "category": {
      "layout": "list",
      "select": true,
      "create": true
    }
  }
}
```

## 基线与租户管理

### 平台配置（PLATFORM）
- 平台定义默认配置
- 所有租户可见
- 租户不能删除或修改平台配置

### 租户配置（TENANT）
- 当 `customizable = true` 时，租户可以完全覆盖配置
- 租户配置仅当前租户可见
- 运行时：租户配置优先于平台配置

### 可见性与定制化
- Client Definition 本身不支持可见性控制
- 通过实体的 `customizable` 字段控制是否允许租户定制客户端配置
