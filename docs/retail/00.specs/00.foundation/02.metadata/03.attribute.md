# Attribute Definition - 属性扩展定义

## 作用

Attribute 定义实体的动态属性扩展，支持复合属性和多语言。

## 核心字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| entityCode | VARCHAR(100) | 是 | - | 关联实体编码（Entity.code） |
| code | VARCHAR(100) | 是 | - | 属性编码（实体内唯一） |
| name | VARCHAR(255) | 是 | - | 属性名称 |
| type | VARCHAR(50) | 是 | - | 属性类型（继承字段类型契约，并额外支持 `composite`） |
| scope | VARCHAR(20) | 是 | PLATFORM | 作用域（PLATFORM/TENANT） |
| customizable | BOOLEAN | 是 | true | 是否允许租户定制 |
| required | BOOLEAN | 是 | false | 是否必填 |
| status | VARCHAR(20) | 是 | ACTIVE | 状态（DRAFT/ACTIVE/DISABLED） |
| isMultilang | BOOLEAN | 是 | false | 是否多语言 |
| parentId | BIGINT | 否 | - | 父属性ID（复合属性） |
| sortOrder | INT | 是 | - | 排序号 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

## 复合属性

`composite` 是 Attribute 专用类型，不属于 Field Type 契约。其作用仅为父属性占位，用于组织子属性结构。

**约束**：
- `type=composite` 的属性不能作为叶子值字段落库
- `type=composite` 必须包含至少一个子属性
- 子属性必须使用字段类型契约中的合法类型（如 `varchar`、`decimal` 等）
- Field 不允许使用 `composite` 类型

```json
{
  "entityCode": "Product",
  "code": "尺寸",
  "name": "尺寸",
  "type": "composite",
  "scope": "PLATFORM",
  "status": "ACTIVE",
  "children": [
    {"code": "长度", "type": "decimal"},
    {"code": "宽度", "type": "decimal"},
    {"code": "高度", "type": "decimal"}
  ]
}
```

## 基线与租户管理

### 平台属性（PLATFORM）
- 平台定义，所有租户可见（除非租户通过 Metadata Customization 覆盖 `status=DISABLED`）
- 租户不能删除或修改类型
- 当 `customizable=true` 时，租户可通过 Metadata Customization 覆盖属性（包括 `status`）

### 租户属性（TENANT）
- 租户自定义属性，仅当前租户可见
- 可以添加租户级别的 Attribute
- 支持复合属性结构

### 可见性控制

**租户级别**（通过 Metadata Customization）：
- 租户可以覆盖平台定义属性的 `status`
- 当 `status=DISABLED` 时属性在该租户下不可见（如果属性的 `customizable=true`）

## 约束规则

1. `entityCode + code + yn` 必须唯一
2. `entityCode` 必须指向有效 `Entity.code`
3. `parentId`（如存在）必须指向同一 `entityCode` 下的 Attribute
4. `scope=TENANT` 的 Attribute 必须归属有效租户上下文
