# Change Request System - 变更请求系统

## 作用

变更请求系统实现实体的审批流程，支持单记录变更、批量变更、多子请求变更等多种场景。系统采用三表模型：主表（用户视角）→ 明细表（系统视角）→ 记录表（数据视角）。

## 三表模型

```
┌─────────────────────────────────────────────────────────────────┐
│                        三表模型                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  1. ChangeRequest（单据主表）                                 │ │
│  │     ├── 用户视角：一个审批单                                  │ │
│  │     ├── 存储审批信息、状态、摘要                              │ │
│  │     └── 1个 ChangeRequest = 1个用户操作                       │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                          │                                       │
│                          │ 1:N                                   │
│                          ▼                                       │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  2. ChangeRequestDetail（单据明细表）                         │ │
│  │     ├── 系统视角：具体的变更明细                               │ │
│  │     ├── 支持：单记录 / 单批量 / 多子请求                       │ │
│  │     └── 1个 ChangeRequest = N个 Detail                        │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                          │                                       │
│                          │ 展开（始终到 record 维度）              │
│                          ▼                                       │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  3. ChangeRequestRecord（记录明细表）                         │ │
│  │     ├── 数据视角：展开到 record 粒度                           │ │
│  │     ├── 所有变更都展开到这里                                   │ │
│  │     └── 支持记录的完整历史追溯                                 │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

## 表结构定义

### 1. ChangeRequest（单据主表）

```sql
CREATE TABLE ChangeRequest (
  -- 主键
  id              VARCHAR(36) PRIMARY KEY,

  -- 类型
  requestType     VARCHAR(10) NOT NULL,     -- SINGLE / BATCH
  entityCode      VARCHAR(100) NOT NULL,
  crType          VARCHAR(10) NOT NULL,     -- U/D/C/BATCH_UPDATE/BATCH_DELETE
  crStatus        VARCHAR(20) NOT NULL,     -- draft/pending/approved/rejected/executing/completed/failed/cancelled

  -- 显示用
  targetDisplay   VARCHAR(500) NOT NULL,    -- "Product#P123" / "品类A批量调价"
  impactCount     INT NOT NULL DEFAULT 1,   -- 影响记录数
  detailCount     INT NOT NULL DEFAULT 1,   -- 明细数量
  changeSummary   VARCHAR(500),             -- 变更摘要

  -- 审批信息
  submittedBy     VARCHAR(100),
  submittedAt     DATETIME,
  approvedBy      VARCHAR(100),
  approvedAt      DATETIME,
  rejectedBy      VARCHAR(100),
  rejectedAt      DATETIME,
  rejectReason    VARCHAR(500),

  -- 执行信息
  executionStatus VARCHAR(20),              -- pending/running/completed/failed
  executedCount   INT DEFAULT 0,
  failedCount     INT DEFAULT 0,

  -- 多级审批（预留）
  currentStage    VARCHAR(50),
  stageHistory    JSON,

  -- 元数据
  yn              CHAR(1) DEFAULT 'N',
  tenantId        VARCHAR(100) NOT NULL,
  createdAt       DATETIME NOT NULL,
  updatedAt       DATETIME,

  INDEX idx_tenant_status (tenantId, crStatus),
  INDEX idx_type_status (requestType, crStatus),
  INDEX idx_created (createdAt DESC),
  INDEX idx_submitted (submittedAt DESC)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 2. ChangeRequestDetail（单据明细表）

```sql
CREATE TABLE ChangeRequestDetail (
  -- 主键
  id              VARCHAR(36) PRIMARY KEY,

  -- 关联主表
  requestId       VARCHAR(36) NOT NULL,
  detailOrder     INT NOT NULL,             -- 明细顺序

  -- 类型
  detailType      VARCHAR(20) NOT NULL,     -- SINGLE / BATCH_FILTER / BATCH_EXPLICIT
  entityCode      VARCHAR(100) NOT NULL,

  -- 筛选条件（BATCH 模式）
  filter          JSON,                     -- {"categoryId": "CAT001"}
  recordIds       JSON,                     -- ["P001", "P002", ...]

  -- 变更规则/数据
  changeData      JSON,                     -- SINGLE: {"price": 110}
  changeRule      JSON,                     -- BATCH: {"price": {"op": "multiply", "value": 1.1}}

  -- 预估
  estimatedCount  INT,

  -- 执行状态
  detailStatus    VARCHAR(20) NOT NULL,     -- pending/running/completed/failed/skipped
  executedCount   INT DEFAULT 0,
  failedCount     INT DEFAULT 0,
  startedAt       DATETIME,
  completedAt     DATETIME,
  failureReason   VARCHAR(500),

  INDEX idx_request (requestId, detailOrder),
  INDEX idx_status (detailStatus),
  INDEX idx_entity (entityCode),

  FOREIGN KEY (requestId) REFERENCES ChangeRequest(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 3. ChangeRequestRecord（记录明细表）

```sql
CREATE TABLE ChangeRequestRecord (
  -- 主键
  id              VARCHAR(36) PRIMARY KEY,

  -- 关联
  requestId       VARCHAR(36) NOT NULL,
  detailId        VARCHAR(36) NOT NULL,     -- 关联到具体明细

  -- 记录标识（核心：始终展开到 record）
  entityCode      VARCHAR(100) NOT NULL,
  recordId        VARCHAR(36) NOT NULL,

  -- 变更内容
  changeType      VARCHAR(10) NOT NULL,     -- UPDATE/DELETE/CREATE
  changeData      JSON NOT NULL,            -- {"price": {"old": 100, "new": 110}}

  -- 变更来源
  sourceType      VARCHAR(10) NOT NULL,     -- DIRECT / INDIRECT
  sourceDesc      VARCHAR(100),             -- "单记录变更" / "品类批量调价"

  -- 状态
  logStatus       VARCHAR(20) NOT NULL,     -- pending/applied/failed/skipped
  appliedAt       DATETIME,
  failureReason   VARCHAR(500),

  -- 快照
  snapshotAt      DATETIME,
  snapshotData    JSON,                     -- 变更时的记录关键属性快照

  -- 时间戳
  createdAt       DATETIME NOT NULL,

  INDEX idx_request (requestId),
  INDEX idx_detail (detailId),
  INDEX idx_record (entityCode, recordId, createdAt DESC),
  INDEX idx_status (logStatus),

  FOREIGN KEY (requestId) REFERENCES ChangeRequest(id) ON DELETE CASCADE,
  FOREIGN KEY (detailId) REFERENCES ChangeRequestDetail(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 审批流程

### 状态流转

```
┌─────────────────────────────────────────────────────────────────┐
│                       状态流转图                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                   │
│  draft ──提交──→ pending ──审批──→ approved ──执行──→ completed │
│   │               │                 │                          │
│   │               │                 └──拒绝──→ rejected          │
│   │               │                                            │
│   └──取消────────┴────────────────────────────→ cancelled       │
│                                                                   │
│  执行失败：                                                     │
│  executing ──失败──→ failed                                     │
│                                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 权限检查

1. **创建变更请求**：检查 `EntityPermission(type=edit/create/delete)`
2. **审批变更请求**：检查 `EntityPermission(type=approve)`
3. **执行变更请求**：审批通过后自动执行

权限控制详见 [实体权限](../../03.permissions/02.entity-permission.md)。

## 展开（Expand）机制

### 何时展开

变更请求在 **审批通过后** 展开：

```
pending ──审批通过──→ approved ──展开──→ executing
                            │
                            ▼
                    ┌───────────────────┐
                    │ 1. 筛选受影响记录   │
                    │ 2. 生成变更数据     │
                    │ 3. 插入 Record 表   │
                    │ 4. 执行变更         │
                    └───────────────────┘
```

### 展开策略

| 规模 | 策略 | 说明 |
|-----|------|------|
| < 1000 条 | 同步展开 | 审批通过后立即展开并执行 |
| >= 1000 条 | 异步展开 | 审批通过后返回，后台异步展开并执行 |

### 展开逻辑

```sql
-- 1. 筛选受影响记录
-- 对于 BATCH_FILTER 类型
SELECT id, categoryId, price, ...
FROM Product
WHERE categoryId = 'CAT001'  -- 从 ChangeRequestDetail.filter 解析
  AND yn = 'N';

-- 2. 生成变更数据
-- 对于 changeRule: {"price": {"op": "multiply", "value": 1.1}}
-- 计算新价格 = 原价格 * 1.1

-- 3. 批量插入 ChangeRequestRecord
INSERT INTO ChangeRequestRecord (
  id, requestId, detailId, entityCode, recordId,
  changeType, changeData, sourceType, sourceDesc,
  snapshotData, logStatus, createdAt
)
SELECT
  UUID(),
  :requestId,
  :detailId,
  'Product',
  p.id,
  'UPDATE',
  JSON_OBJECT('price', JSON_OBJECT('old', p.price, 'new', p.price * 1.1)),
  'INDIRECT',
  '品类批量调价+10%',
  JSON_OBJECT('categoryId', p.categoryId, 'price', p.price),
  'pending',
  NOW()
FROM Product p
WHERE p.categoryId = 'CAT001'
  AND p.yn = 'N';

-- 4. 更新 ChangeRequestDetail 状态
UPDATE ChangeRequestDetail
SET detailStatus = 'completed',
    executedCount = :executedCount,
    failedCount = :failedCount,
    completedAt = NOW()
WHERE id = :detailId;
```

## 查询示例

### 查询列表（所有变更请求）

```sql
SELECT
  cr.id,
  cr.requestType,
  cr.targetDisplay,
  cr.impactCount,
  cr.detailCount,
  cr.crStatus,
  cr.changeSummary,
  cr.submittedBy,
  cr.submittedAt
FROM ChangeRequest cr
WHERE cr.tenantId = :tenantId
  AND cr.yn = 'N'
  AND (:status IS NULL OR cr.crStatus = :status)
ORDER BY cr.submittedAt DESC
LIMIT :pageSize OFFSET :offset;
```

### 查询详情（含明细）

```sql
SELECT
  cr.*,
  crd.id AS detailId,
  crd.detailOrder,
  crd.detailType,
  crd.filter,
  crd.changeRule,
  crd.estimatedCount,
  crd.executedCount,
  crd.detailStatus
FROM ChangeRequest cr
INNER JOIN ChangeRequestDetail crd ON crd.requestId = cr.id
WHERE cr.id = :requestId
ORDER BY crd.detailOrder;
```

### 查询某个记录的完整变更历史

```sql
-- 查询 Product#P123 的所有变更（包括直接和间接）
SELECT
  crr.id,
  crr.requestId,
  cr.targetDisplay,
  crr.detailId,
  crd.detailType,
  crr.sourceType,
  crr.sourceDesc,
  crr.changeData,
  crr.snapshotData,
  crr.logStatus,
  crr.appliedAt,
  crr.createdAt
FROM ChangeRequestRecord crr
INNER JOIN ChangeRequest cr ON cr.id = crr.requestId
INNER JOIN ChangeRequestDetail crd ON crd.id = crr.detailId
WHERE crr.entityCode = 'Product'
  AND crr.recordId = 'P123'
ORDER BY crr.createdAt DESC;
```

### 查询多品类批量变更的执行进度

```sql
SELECT
  cr.id AS requestId,
  cr.targetDisplay,
  cr.impactCount,
  cr.detailCount,
  cr.crStatus,
  crd.id AS detailId,
  crd.detailOrder,
  crd.detailType,
  crd.estimatedCount,
  crd.executedCount,
  crd.failedCount,
  crd.detailStatus
FROM ChangeRequest cr
INNER JOIN ChangeRequestDetail crd ON crd.requestId = cr.id
WHERE cr.id = :requestId
ORDER BY crd.detailOrder;
```

## 约束规则

1. **请求存在性**：`ChangeRequestDetail.requestId` 必须引用有效的 `ChangeRequest.id`
2. **明细唯一性**：`ChangeRequestDetail.id` 必须唯一
3. **记录关联性**：`ChangeRequestRecord.detailId` 必须引用有效的 `ChangeRequestDetail.id`
4. **状态一致性**：`ChangeRequest.crStatus` 需与所有 `ChangeRequestDetail.detailStatus` 协调更新
5. **展开完整性**：审批通过后，所有受影响记录必须展开到 `ChangeRequestRecord`
