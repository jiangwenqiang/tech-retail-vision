# Runtime Specification - 运行时规范

## 作用

运行时规范定义系统在运行时的核心机制和数据模型，包括变更请求系统、审计日志、任务调度等。与元数据规范（定义结构）和权限规范（定义访问控制）共同构成完整的技术规范体系。

## 设计原则

### 分层清晰

- **元数据层**：定义"是什么"（实体结构、字段类型、关系）
- **权限层**：定义"谁能操作"（角色、权限、范围）
- **运行时层**：定义"怎么执行"（审批流程、审计日志、任务调度）

### 关注点分离

| 关注点 | 所属规范 | 示例 |
|-------|---------|------|
| 实体有哪些字段 | 元数据规范 | Entity.code, Field.type |
| 谁能修改价格 | 权限规范 | EntityPermission(type=edit) |
| 价格修改需要审批吗 | 元数据规范 | Entity.approvalConfig |
| 审批流程怎么执行 | 运行时规范 | ChangeRequest 三表模型 |
| 谁能审批 | 权限规范 | EntityPermission(type=approve) |
| 审批历史如何追溯 | 运行时规范 | RecordChangeLog |

### 数据一致性

- 变更请求系统与实体元数据的 `approvalConfig` 配置对齐
- 审计日志系统与实体元数据的 `auditEnabled` 配置对齐
- 权限检查贯穿所有运行时操作
