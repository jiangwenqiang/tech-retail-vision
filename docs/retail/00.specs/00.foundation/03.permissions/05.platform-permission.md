# Platform Permission - 平台权限

## 作用

Platform Permission 定义平台角色对系统级操作的访问权限。与租户权限（EntityPermission / ScopePermission / FieldPermission）不同，平台权限控制的是系统管理操作（元数据管理、租户管理、系统配置），而非业务数据 CRUD。

平台权限仅适用于 `scope=PLATFORM` 的固定角色，不支持自定义。

## 权限格式

采用 `resource:action` 格式，支持 `*` 通配符：

| 格式 | 说明 | 示例 |
|-----|------|------|
| `resource:action` | 精确匹配 | `metadata:entity:create` |
| `resource:*` | 匹配某资源的所有操作 | `metadata:entity:*` |
| `*:action` | 匹配所有资源的某操作 | `*:read` |
| `*:*` | 匹配所有权限 | PLATFORM_ADMIN 使用 |

### 通配符匹配规则

```
请求权限:  metadata:entity:create

角色权限                    匹配结果
metadata:*:*               → ✅ 匹配
metadata:entity:*          → ✅ 匹配
metadata:entity:create     → ✅ 匹配
metadata:field:*           → ❌ 不匹配
tenant:create              → ❌ 不匹配
```

匹配优先级：精确匹配 > 资源通配 > 全通配。

## 权限定义

### 元数据权限（metadata:*）

控制元数据定义的创建、修改和发布。

#### metadata:entity — 实体定义

| 操作 | 代码 | 说明 |
|-----|------|------|
| 创建 | metadata:entity:create | 创建新的实体定义 |
| 读取 | metadata:entity:read | 查看实体定义 |
| 修改 | metadata:entity:update | 修改实体定义属性 |
| 删除 | metadata:entity:delete | 删除实体定义 |

#### metadata:field — 字段定义

| 操作 | 代码 | 说明 |
|-----|------|------|
| 创建 | metadata:field:create | 创建字段定义 |
| 读取 | metadata:field:read | 查看字段定义 |
| 修改 | metadata:field:update | 修改字段定义属性 |
| 删除 | metadata:field:delete | 删除字段定义 |

#### metadata:attribute — 属性定义

| 操作 | 代码 | 说明 |
|-----|------|------|
| 创建 | metadata:attribute:create | 创建属性定义 |
| 读取 | metadata:attribute:read | 查看属性定义 |
| 修改 | metadata:attribute:update | 修改属性定义属性 |
| 删除 | metadata:attribute:delete | 删除属性定义 |

#### metadata:release — 版本发布

| 操作 | 代码 | 说明 |
|-----|------|------|
| 创建 | metadata:release:create | 创建发布包 |
| 读取 | metadata:release:read | 查看发布包详情 |
| 审批 | metadata:release:approve | 审批发布（允许执行上线） |
| 回滚 | metadata:release:rollback | 回滚已执行的发布 |

> 发布的执行（apply）由系统在审批通过后自动触发，不需要单独的执行权限。

#### metadata:tenant — 租户级元数据

| 操作 | 代码 | 说明 |
|-----|------|------|
| 定制 | metadata:tenant:customize | 通过三模型定制入口修改租户元数据 |
| 提升 | metadata:tenant:promote | 将租户定制提升为平台基线 |

### 租户管理权限（tenant:*）

控制租户生命周期管理。与 `metadata:tenant:*`（租户元数据操作）不同，`tenant:*` 控制的是租户账户本身的管理。

| 操作 | 代码 | 说明 |
|-----|------|------|
| 创建 | tenant:create | 创建新租户 |
| 读取 | tenant:read | 查看租户信息 |
| 修改 | tenant:update | 修改租户配置（套餐、限额等） |
| 启用 | tenant:enable | 启用已停用的租户 |
| 停用 | tenant:disable | 停用租户（不可登录，数据保留） |

> 租户删除为高风险操作，暂不开放 API 入口，需通过平台运维流程处理。

### 系统管理权限（system:*）

控制系统配置和监控。

| 操作 | 代码 | 说明 |
|-----|------|------|
| 配置 | system:config | 修改系统级配置 |
| 监控 | system:monitor | 查看系统监控数据 |
| 审计 | system:audit | 查看审计日志 |

## 平台角色权限映射

### PLATFORM_ADMIN（平台管理员）

```
*:*
```

拥有全部平台操作权限，以及所有租户内数据的完全访问权限（穿透 EntityPermission / ScopePermission / FieldPermission）。

| 权限 | 说明 |

## 权限检查规则

### 检查流程

```
用户请求平台操作
        │
        ▼
  ┌──────────────────┐
  │ 平台角色检查       │ ── 非平台角色 ──→ 403
  │ scope=PLATFORM    │
  └─────┬────────────┘
        │ 是平台角色
        ▼
  ┌──────────────────┐
  │ 权限匹配           │ ── 无匹配 ──→ 403
  │ resource:action   │
  └─────┬────────────┘
        │ 匹配（含通配符）
        ▼
  ┌──────────────────┐
  │ ABAC 约束检查      │ ── 违反约束 ──→ 403
  │（租户禁止修改      │
  │  平台基线等）      │
  └─────┬────────────┘
        │ 通过
        ▼
     执行操作
```

### 固定角色约束

平台角色及其权限映射由系统初始化时写入，不支持：

- 创建新的平台角色（`scope=PLATFORM` 的角色不能通过 API 创建）
- 修改固定角色的权限映射
- 删除固定角色

## 平台角色穿透

平台角色访问租户内业务数据时，穿透租户权限体系的三层检查：

| 层 | 穿透行为 |
|---|---------|
| EntityPermission | 穿透，等同于所有实体所有操作均为 `level=yes` |
| ScopePermission | 穿透，不过滤任何记录（可访问租户所有数据） |
| FieldPermission | 穿透，所有字段均可读写（不脱敏、不隐藏） |

### 穿透条件

- 必须显式指定目标租户上下文（通过请求参数或 header 传入 `tenantId`）
- 穿透操作记录审计日志，包含操作者、目标租户、操作内容

### 非穿透场景

以下场景不适用穿透规则，仍需对应平台权限：

| 场景 | 需要的权限 |
|-----|---------|
| 修改租户配置 | `tenant:update` |
| 查看租户监控 | `system:monitor` |
| 查看审计日志 | `system:audit` |
| 管理租户角色 | `tenant:update` |

## ABAC 补充约束

平台权限通过后，仍需满足以下强制性约束（面向租户角色时由系统自动拦截）：

1. **租户禁止基线变更**：租户角色（scope=TENANT）不能执行 `metadata:entity`/`field`/`attribute` 的 create/update/delete
2. **定制门控**：租户仅可通过三模型定制入口（Metadata Customization / Client Customization / Layout Profile）修改 `customizable=true` 的对象
3. **租户属性隔离**：平台属性仅允许覆盖白名单字段；租户自建属性（scope=TENANT）仅限本租户操作
4. **Client/Layout 基线保护**：不允许直接修改平台基线的 Client/Layout 记录

## 与租户权限的对比

| 维度 | 平台权限 | 租户权限 |
|-----|---------|---------|
| 格式 | `resource:action` | Role → EntityPermission / ScopePermission / FieldPermission |
| 角色 | 3 个固定角色 | 固定角色 + 自定义角色 |
| 可配置性 | 不可配置（代码定义） | 可配置（数据库存储） |
| 控制对象 | 系统操作 | 业务数据 |
| 检查顺序 | 先于租户权限 | 平台权限通过后检查 |
| 通配符 | 支持 `*` | 不支持 |
| 数据范围 | 通过穿透覆盖所有数据 | 通过 ScopePermission 精确控制 |
