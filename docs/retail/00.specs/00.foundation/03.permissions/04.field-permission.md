# Field Permission - 字段权限
## 作用
Field Permission 控制角色对某个实体中特定字段的操作权限。采用行式设计，每种操作类型（type）独立一条记录，通过 `props` 字段携带类型特定的参数。
## 核心字段
| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| entityCode | VARCHAR(100) | 是 | 实体编码（Entity.code） |
| roleCode | VARCHAR(100) | 是 | 角色编码（Role.code） |
| fieldCode | VARCHAR(100) | 是 | 字段编码（Field.code 或 Attribute.code） |
| type | VARCHAR(20) | 是 | 权限类型（read/write） |
| level | VARCHAR(20) | 是 | 权限级别（yes/no/masked） |
| props | JSON | 否 | 类型特定参数 |
| yn | CHAR(1) | 是 | N | 软删除标记 |
| entityCode | roleCode | fieldCode | read   | write |
|------------|----------|-----------|--------|-------|
| Product    | Manager  | price     | yes    | yes   |
```
行式设计将每种操作类型独立一条记录：
```
| entityCode | roleCode | fieldCode | type  | level  | props           |
|------------|----------|-----------|-------|--------|-----------------|
| Product    | Manager  | price     | read  | yes    | null            |
| Product    | Manager  | price     | write | yes    | null            |
| Product    | Salesman | price     | read  | masked | {"strategy":"amount"} |
| Product    | Salesman | price     | write | no     | null            |
```
优势：
## 权限类型（type）
| 类型 | 代码 | 说明 |
|-----|------|------|
| 读取 | read | 查询时字段的可见性与展示方式 |
| 写入 | write | 创建/编辑时字段的可写性 |
## 权限级别（level）
### read 类型
| 级别 | 说明 | props |
|-----|------|-------|
| yes | 返回完整值 | 无 |
| no | 字段不可见（响应中不包含该字段） | 无 |
| masked | 返回脱敏值 | `{"strategy": "amount"}` |
### write 类型
| 级别 | 说明 | props |
|-----|------|-------|
| yes | 可写入 | 无 |
| no | 不可写入（编辑时拒绝，创建时静默丢弃） | 无 |
### read 与 write 的关系
- `read=no` 的字段，`write` 无意义（创建时静默丢弃，编辑时拒绝）
- `read=masked` 的字段，`write` 仅允许 `no`（不可写脱敏值）
- `write=no` 的字段，在编辑表单中可见但不可修改（只读）
## props 字段
`props` 承载 `type` 特定的补充参数，不同 `type` 的 props 契约不同。
### type=read 的 props
| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| strategy | VARCHAR(30) | 当 level=masked 时必填 | 脱敏策略 |
预置脱敏策略：
| 策略 | 说明 | 原始值 | 脱敏后 |
|-----|------|--------|-------|
| phone | 手机号（保留前3后4） | 13812345678 | `138****5678` |
| name | 姓名（保留姓） | 张三丰 | `张**` |
| email | 邮箱（保留首字母和域名） | zhang@example.com | `z***@example.com` |
| idCard | 身份证（保留前3后4） | 310101199001011234 | `310***********1234` |
| amount | 金额完全隐藏 | 50.00 | `***.**` |
| hidden | 完全隐藏 | 任意值 | `***` |
### type=write 的 props
当前无参数，预留扩展。未来可支持：
| 参数（预留） | 说明 |
|-----|------|
| requireReason | 修改时是否需要填写原因 |
| requireApproval | 修改时是否需要审批 |
## 配置示例
### 商品进价 — 不同角色的完整配置
```json
// 店长：完整可读可写
{
  "entityCode": "Product",
  "roleCode": "STORE_MANAGER",
  "fieldCode": "basePurchasePrice",
  "type": "read",
  "level": "yes",
  "props": null
}
{
  "entityCode": "Product",
  "roleCode": "STORE_MANAGER",
  "fieldCode": "basePurchasePrice",
  "type": "write",
  "level": "yes",
  "props": null
}
// 销售员：脱敏可读，不可写
{
  "entityCode": "Product",
  "roleCode": "SALESMAN",
  "fieldCode": "basePurchasePrice",
  "type": "read",
  "level": "masked",
  "props": { "strategy": "amount" }
}
{
  "entityCode": "Product",
  "roleCode": "SALESMAN",
  "fieldCode": "basePurchasePrice",
  "type": "write",
  "level": "no",
  "props": null
}
// 收银员：完全不可见
{
  "entityCode": "Product",
  "roleCode": "CASHIER",
  "fieldCode": "basePurchasePrice",
  "type": "read",
  "level": "no",
  "props": null
}
```
### 客户手机号 — 仅管理员可改
```json
{
  "entityCode": "Customer",
  "roleCode": "SERVICE_STAFF",
  "fieldCode": "phoneNumber",
  "type": "read",
  "level": "masked",
  "props": { "strategy": "phone" }
}
{
  "entityCode": "Customer",
  "roleCode": "SERVICE_STAFF",
  "fieldCode": "phoneNumber",
  "type": "write",
  "level": "no",
  "props": null
}
```
## 零配置行为
当角色没有为某个字段的某个 type 配置 FieldPermission 时：
| 场景 | 默认行为 |
|-----|---------|
| type=read 未配置 | level=yes（全开） |
| type=write 未配置 | level=yes（全开） |
| 整个字段无任何 FieldPermission | 所有 type 均默认全开 |
即：**字段权限默认全开，仅在显式配置时才限制。**
## 适用范围
FieldPermission 同时适用于：
| 对象类型 | fieldCode 引用 | 说明 |
|---------|---------------|------|
| Entity 基础字段 | Field.code | 实体定义中的字段 |
| 租户扩展属性 | Attribute.code | 租户自定义的属性 |
## API 行为
### 查询请求
| type=read level | 列表查询 | 详情查询 | 导出 | 搜索/过滤 |
|----------------|---------|---------|------|----------|
| yes | 返回完整值 | 返回完整值 | 导出完整值 | 可按该字段过滤 |
| masked | 返回脱敏值 | 返回脱敏值 | 导出脱敏值 | 不可按该字段过滤 |
| no | 不含该字段 | 不含该字段 | 不含该列 | 不可按该字段过滤 |
```json
// 店长请求（read level=yes）
GET /api/products/123
{
  "productName": "商品A",
  "basePurchasePrice": 50.00
}
// 销售员请求（read level=masked, strategy=amount）
GET /api/products/123
{
  "productName": "商品A",
  "basePurchasePrice": "***.**"
}
// 收银员请求（read level=no）
GET /api/products/123
{
  "productName": "商品A"
}
```
### 写入请求
| type=write level | 创建请求包含该字段 | 编辑请求包含该字段 |
|-----------------|------------------|------------------|
| yes | 正常写入 | 正常写入 |
| no | 静默丢弃，不报错 | 返回 403 拒绝 |
```json
// 销售员尝试修改进价（write level=no）
PUT /api/products/123
{
  "basePurchasePrice": 60.00
}
// 响应
{
  "status": 403,
  "message": "Field 'basePurchasePrice' is not writable for role 'SALESMAN'"
}
```
### 聚合查询
对于需要参与计算但不展示原始值的场景，提供显式的聚合接口：
```http
GET /api/products/123/profit-margin
```
服务端用完整值计算后返回结果，不暴露原始进价：
```json
{
  "salePrice": 79.00,
  "profitMargin": "36.7%"
}
```
## 敏感字段管控
### 敏感字段标记
在 Entity 的 Field 定义中，通过 `sensitive` 属性标记：
```json
{
  "code": "basePurchasePrice",
  "name": "采购进价",
  "type": "decimal",
  "sensitive": true,
  "params": {
    "precision": 10,
    "scale": 2
  }
}
```
`sensitive=true` 的字段会被系统默认加密存储（参见[安全架构](../../01.architecture/00.foundation/02.metadata/04.security/authorization.md)中的敏感数据处理）。
### 审计日志
对敏感字段的访问记录审计日志（无论 level 是 yes 还是 masked）：
```json
{
  "eventType": "sensitive_field.access",
  "userId": "user_123",
  "resourceType": "Product",
  "resourceId": "456",
  "field": "basePurchasePrice",
  "accessLevel": "masked",
  "timestamp": "2024-03-15T10:30:00Z"
}
```
## 多角色合并
当用户绑定多个角色时，**同一 type 的 level 取并集**，不同 type 独立合并：
### 按 type 独立合并
```
// type=read
角色 A: basePurchasePrice → level=yes
角色 B: basePurchasePrice → level=masked
合并结果: level=yes（取更宽松的）
// type=write
角色 A: basePurchasePrice → level=no
角色 B: basePurchasePrice → level=no
合并结果: level=no
```
### read type 的合并优先级
| 角色 A | 角色 B | 合并结果 | 说明 |
|--------|--------|---------|------|
| yes | no | yes | 有任一角色授权即取完整值 |
| yes | masked | yes | 同上 |
| masked | no | masked | 无完全授权，保持脱敏 |
| masked | masked | masked | 保持脱敏 |
| no | no | no | 完全不可见 |
核心原则：`yes > masked > no`
### write type 的合并优先级
`yes > no`（存在 yes 即 yes）
### props 合并
当多个角色合并后取到 `level=yes` 时，props 无意义（返回完整值）。
当合并后取到 `level=masked` 时，如果多个角色的 props.strategy 不同，取第一个匹配的（按角色 level 排序）。
## 与只读字段的交互
| 机制 | 作用 | 适用范围 |
|-----|------|---------|
| Field.readOnly | 元数据定义，适用于所有用户 | 不可通过 FieldPermission 覆盖 |
| FieldPermission (type=write, level=no) | 权限控制，仅适用于特定角色 | 不影响其他角色 |
- `Field.readOnly=true`：所有用户不可写（系统级保护）
- `FieldPermission (type=write, level=no)`：仅该角色不可写（权限级控制）
- 两者同时生效时取更严格的结果
## 约束规则
1. **实体存在性**：`entityCode` 必须引用有效的 `Entity.code`
2. **角色存在性**：`roleCode` 必须引用有效的 `Role.code`
3. **字段存在性**：`fieldCode` 必须引用同一 `entityCode` 下的有效 `Field.code` 或 `Attribute.code`
4. **绑定唯一性**：`entityCode + roleCode + fieldCode + type + yn` 必须唯一
5. **type 值域**：`type` 必须为 `read` 或 `write`
6. **level 值域**：type=read 时 level 必须为 `yes`、`no`、`masked`；type=write 时 level 必须为 `yes`、`no`
7. **props 契约**：`props` 必须符合对应 `type + level` 的参数白名单
8. **masked 必须带 strategy**：level=masked 时 props 中必须包含有效的 strategy
## 扩展点
### 预留的 type 值
| type | 说明 | 预留 props |
|------|------|-----------|
| export | 控制是否可导出该字段 | `{ "level": "yes"/"no" }` |
| search | 控制是否可按该字段搜索 | `{ "level": "yes"/"no" }` |
### 自定义脱敏策略
租户可通过配置注册自定义 maskStrategy handler，实现业务特定的脱敏逻辑。
