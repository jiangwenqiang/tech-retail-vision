# Role Definition - 角色定义

## 作用

Role 是权限的基本载体单元。每个角色包含一组实体权限（EntityPermission）、范围权限（ScopePermission）和字段权限（FieldPermission）。用户通过绑定一个或多个角色获得相应权限。

## 结构组成

```
Role
├── props（基础属性）
├── entityPermissions（实体权限）
├── scopePermissions（范围权限）
└── fieldPermissions（字段权限）
```

## props（基础属性）

### 核心字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| code | VARCHAR(100) | 是 | - | 角色编码（租户内唯一） |
| name | VARCHAR(255) | 是 | - | 角色名称 |
| type | VARCHAR(20) | 是 | CUSTOM | 角色类型 |
| level | INT | 是 | 0 | 角色层级（数值越大优先级越高） |
| scope | VARCHAR(20) | 是 | TENANT | 作用域（PLATFORM/TENANT） |
| isEnabled | BOOLEAN | 是 | true | 是否启用 |
| isDefault | BOOLEAN | 是 | false | 新用户是否默认绑定 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

### 角色类型（type）

| 类型 | 代码 | 说明 | 可修改 | 可删除 |
|-----|------|------|-------|-------|
| 固定角色 | FIXED | 系统预设角色 | 否 | 否 |
| 自定义角色 | CUSTOM | 租户创建的业务角色 | 是 | 是 |

### 预设角色

**平台固定角色**（scope=PLATFORM）：

| 角色编码 | 名称 | 说明 | 权限范围 |
|---------|------|------|---------|
| PLATFORM_ADMIN | 平台管理员 | 平台最高权限 | 所有平台操作 + 租户内完全访问 |
| PLATFORM_ARCHITECT | 平台架构师 | 元数据架构管理 | 元数据管理 + 发布审批 |
| PLATFORM_DEVELOPER | 平台开发者 | 平台开发 | 元数据读写 |

> 平台角色的具体权限列表参见 [平台权限](./05.platform-permission.md)。

**租户固定角色**（scope=TENANT）：

| 角色编码 | 名称 | 说明 | 数据范围 |
|---------|------|------|---------|
| TENANT_ADMIN | 租户管理员 | 租户最高权限 | all（全部数据） |
| TENANT_DEVELOPER | 租户开发者 | 元数据定制 | all |
| TENANT_USER | 租户用户 | 基础用户 | own |

### 角色示例

```json
{
  "code": "STORE_MANAGER",
  "name": "店长",
  "type": "CUSTOM",
  "level": 50,
  "scope": "TENANT",
  "isEnabled": true,
  "isDefault": false
}
```

## 角色层级（level）

角色层级用于多角色合并时的优先级判定：

| 场景 | 规则 |
|-----|------|
| 实体权限冲突 | 任一角色允许即允许（并集） |
| 数据范围冲突 | 取最宽范围（all > organization > own > none） |
| 字段权限冲突 | 任一角色允许即允许（并集） |
| 范围权限 | 所有角色的过滤条件取交集 |
| level 相同 | 角色编码字典序排序 |

> 注：`level` 主要用于管理界面展示排序和复杂场景下的仲裁参考，权限合并不依赖 level 计算。

## 角色分配

### UserRoleBinding（用户-角色绑定）

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| userId | VARCHAR(100) | 是 | 用户 ID |
| roleCode | VARCHAR(100) | 是 | 角色编码 |
| tenantId | VARCHAR(100) | 是 | 租户 ID |
| assignedBy | VARCHAR(100) | 是 | 分配者用户 ID |
| assignedAt | DATETIME | 是 | 分配时间 |
| yn | CHAR(1) | 是 | N | 软删除标记 |

### 约束规则

1. **角色归属**：`scope=PLATFORM` 的角色只能分配给平台用户；`scope=TENANT` 的角色只能分配给租户用户
2. **固定角色不可删除**：`type=FIXED` 的角色不可删除或修改基础属性
3. **固定角色不可取消绑定**：`type=FIXED` 的角色绑定关系不可删除（如 TENANT_ADMIN 绑定关系不可移除）
4. **编码唯一性**：`code + scope + yn` 必须唯一
5. **绑定唯一性**：`userId + roleCode + tenantId + yn` 必须唯一
6. **租户隔离**：`scope=TENANT` 的角色仅在创建该角色的租户内可见和可用
7. **层级约束**：租户管理员角色（TENANT_ADMIN）的 `level` 值最高，自定义角色的 `level` 不可超过

## 角色与权限对象的关系

```
Role (1) ──< (N) EntityPermission
Role (1) ──< (N) ScopePermission
Role (1) ──< (N) FieldPermission
Role (N) >──< (N) User（通过 UserRoleBinding）
```
