# Entity Permission - 实体权限
## 作用
Entity Permission 定义角色对某个实体类型的操作权限。采用行式设计，每种操作类型（type）独立一条记录，通过 `level` 控制权限级别，通过 `props` 携带类型特定的参数。
## 核心字段
| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| entityCode | VARCHAR(100) | 是 | 实体编码（Entity.code） |
| roleCode | VARCHAR(100) | 是 | 角色编码（Role.code） |
| type | VARCHAR(20) | 是 | 操作类型（read/create/edit/delete） |
| level | VARCHAR(20) | 是 | 权限级别（yes/no） |
| props | JSON | 否 | 类型特定参数 |
| yn | CHAR(1) | 是 | N | 软删除标记 |
| entityCode | roleCode | read | create | edit | delete |
|------------|----------|------|--------|------|--------|
| Product    | Manager  | yes  | yes    | yes  | no     |
```
行式设计将每种操作类型独立一条记录：
```
| entityCode | roleCode | type   | level | props                          |
|------------|----------|--------|-------|--------------------------------|
| Product    | Manager  | read   | yes   | null                           |
| Product    | Manager  | create | yes   | null                           |
| Product    | Manager  | edit   | yes   | null                           |
| Product    | Manager  | delete | no    | null                           |
| Order      | Salesman | edit   | own   | {"allowAssignTo": "self"}      |
```

## 操作类型与权限级别
### type=read
| level | 说明 | props |
|-------|------|-------|
| yes | 可读取该实体（可见记录集由 ScopePermission 决定） | 无 |
| no | 不可读取任何记录 | 无 |
> read 的 level 仅有 yes/no 两个值。数据范围控制已统一到 [ScopePermission](./03.scope-permission.md)，通过 `type=scope` 的过滤器实现。
### type=create
| level | 说明 | props |
|-------|------|-------|
| yes | 可创建新记录 | 无 |
| no | 不可创建 | 无 |
### type=edit
| level | 说明 | props |
|-------|------|-------|
| yes | 可编辑可见记录集中的所有记录 | 无 |
| no | 不可编辑任何记录 | 无 |
| own | 仅可编辑可见记录集中由自己创建或分配给自己的记录 | `{ "allowAssignTo": "self"|"none"|"team" }` |
### type=delete
| level | 说明 | props |
|-------|------|-------|
| yes | 可删除可见记录集中的所有记录 | `{ "softOnly": true }` |
| no | 不可删除任何记录 | 无 |
| own | 仅可删除可见记录集中由自己创建或分配给自己的记录 | `{ "softOnly": true }` |
## 权限判定层级
```
EntityPermission(type=read, level=no)   → 拒绝，不进入后续检查
EntityPermission(type=read, level=yes)  → 进入 ScopePermission 检查
  └── ScopePermission(type=scope)           → 组织级/所有权级记录过滤
  └── ScopePermission(type=field/expression/callback) → 追加条件过滤
       └── EntityPermission(type=edit, level=yes/own) → 在可见记录集上操作
            └── FieldPermission → 字段裁剪
```
**核心关系**：EntityPermission 回答"能不能操作这个实体"，ScopePermission 回答"能操作哪些记录"，两者各司其职。
## props 字段
### type=edit 的 props
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| allowAssignTo | VARCHAR(20) | 否 | self | 当 level=own 时，编辑操作是否允许转移所有权 |
| allowAssignTo | 说明 |
|--------------|------|
| self | 可将记录分配给自己（默认） |
| none | 不可更改记录的分配关系 |
| team | 可将记录分配给自己所属组织的成员（预留） |
### type=delete 的 props
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|-----|------|------|-------|------|
| softOnly | BOOLEAN | 否 | true | 是否仅允许软删除 |
| softOnly | 说明 |
|----------|------|
| true | 仅允许软删除（设 yn=Y），不允许物理删除（默认） |
| false | 允许软删除和物理删除 |
## 配置示例
### 基础示例
```json
// 店长对 Product 的权限
[
  { "entityCode": "Product", "roleCode": "STORE_MANAGER", "type": "read",   "level": "yes", "props": null },
  { "entityCode": "Product", "roleCode": "STORE_MANAGER", "type": "create", "level": "yes", "props": null },
  { "entityCode": "Product", "roleCode": "STORE_MANAGER", "type": "edit",   "level": "yes", "props": null },
  { "entityCode": "Product", "roleCode": "STORE_MANAGER", "type": "delete", "level": "no",  "props": null }
]
```
含义：店长可以读取产品（可见范围由 ScopePermission 决定）、创建和编辑产品，但不能删除。
### 销售员看自己的订单
```json
// 销售员对 Order 的权限
[
  { "entityCode": "Order", "roleCode": "SALESMAN", "type": "read",   "level": "yes", "props": null },
  { "entityCode": "Order", "roleCode": "SALESMAN", "type": "create", "level": "yes", "props": null },
  { "entityCode": "Order", "roleCode": "SALESMAN", "type": "edit",   "level": "own", "props": { "allowAssignTo": "self" } },
  { "entityCode": "Order", "roleCode": "SALESMAN", "type": "delete", "level": "no",  "props": null }
]
// 配合 ScopePermission（type=scope, level=own）实现"仅看自己的订单"
```
含义：销售员可创建订单，可查看和编辑自己的订单（但不可转移分配），不可删除。
### 完全禁止
```json
[
  { "entityCode": "Product", "roleCode": "SALESMAN", "type": "read",   "level": "no", "props": null },
  { "entityCode": "Product", "roleCode": "SALESMAN", "type": "create", "level": "no", "props": null },
  { "entityCode": "Product", "roleCode": "SALESMAN", "type": "edit",   "level": "no", "props": null },
  { "entityCode": "Product", "roleCode": "SALESMAN", "type": "delete", "level": "no", "props": null }
]
```
### 删除仅允许软删除
```json
{
  "entityCode": "Product",
  "roleCode": "STORE_MANAGER",
  "type": "delete",
  "level": "yes",
  "props": { "softOnly": true }
}
```
## 零配置默认值
当角色没有为某个实体的某个 type 配置 EntityPermission 时：
| 实体类型 | 默认 read | 默认 create | 默认 edit | 默认 delete |
|---------|----------|------------|----------|------------|
| PLATFORM 实体 | no | no | no | no |
| TENANT 实体（实体 scope=PLATFORM） | yes | no | no | no |
| TENANT 实体（实体 scope=TENANT） | yes | yes | yes | yes |
> 零配置默认值仅适用于 `type=CUSTOM` 的角色。固定角色（`type=FIXED`）的权限由系统初始化时写入。
## 多角色合并
同一 type 的 level 取并集：
```
// type=edit
角色 A: Product → level=own
角色 B: Product → level=yes
合并结果: level=yes（取更宽松的）
```
合并优先级：`yes > own > no`
props 合并：
- **level=yes 时**：edit/delete 的 props 无意义（yes 不受 own 限制），丢弃
- **level=own 时**：allowAssignTo 取更宽（`team > self > none`）
- **type=delete**：softOnly 取 false 优先（有任一角色允许物理删除即允许）
## 约束规则
1. **实体存在性**：`entityCode` 必须引用有效的 `Entity.code`，且该实体 `status=ACTIVE`
2. **角色存在性**：`roleCode` 必须引用有效的 `Role.code`，且在同一 `scope` 内
3. **绑定唯一性**：`entityCode + roleCode + type + yn` 必须唯一
4. **type 值域**：`type` 必须为 `read`、`create`、`edit`、`delete` 之一
5. **level 值域**：type=read/create 时 level 必须为 `yes`、`no`；type=edit/delete 时 level 必须为 `yes`、`no`、`own`
6. **props 契约**：`props` 必须符合对应 `type` 的参数白名单
## 扩展点
### 预留的 type 值
| type | 说明 | 预留 props |
|------|------|-----------|
| export | 控制是否可导出该实体数据 | `{ "maxRows": 10000 }` |
| import | 控制是否可导入数据到该实体 | `{ "allowedFormats": ["csv", "xlsx"] }` |
| approve | 控制是否可审批该实体的记录 | `{ "stages": ["draft", "pending"] }` |
新增 type 不需要改表结构，只需加 type 枚举值和对应 level/props 契约。
|---------|-------|------|
| `scopes.Account.read = "yes"` | `type=read, level=yes` | read 本身只有 yes/no，数据范围由 ScopePermission 控制 |
| `scopes.Account.edit = "own"` | `type=edit, level=own` | edit/delete 保留 own 级别 |
| `scopes.Account.acl = true` | 无对应 | 本方案中所有实体默认参与权限控制 |
| Team 数据隔离 | ScopePermission(type=scope) | 通过统一的 ScopePermission 管理 |
