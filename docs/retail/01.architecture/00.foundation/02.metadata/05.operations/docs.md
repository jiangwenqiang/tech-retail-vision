# 元数据运维架构

## 概述

运维架构定义元数据管理系统的运维体系，包括监控、告警、部署、备份和故障处理，确保系统稳定运行。

## 子域目录

1. [监控与告警](./monitoring.md)
   - 监控体系（应用、中间件、基础设施）
   - 应用指标（性能、业务、系统）
   - Prometheus 配置
   - 链路追踪（Jaeger）
   - 日志聚合（ELK）
   - 告警配置
   - 告警通知
   - 部署架构
   - 故障处理
   - 备份与恢复
   - 运维自动化

## 核心监控指标

### 性能指标
- 请求响应时间（P95 < 200ms）
- 请求速率
- 错误率（< 0.1%）
- 缓存命中率（> 80%）
- 数据库查询时间（P95 < 50ms）

### 业务指标
- 实体总数
- 字段总数
- 属性总数
- 业务唯一冲突次数
- `props_json` 解析失败次数
- 租户自定义扩展数

### 系统指标
- CPU 使用率（> 80% 告警）
- 内存使用率（> 85% 告警）
- 磁盘使用率（> 80% 告警）
- 线程数（> 500 告警）
- GC 时间（P95 > 1s 告警）

## 故障等级

| 等级 | 描述 | 响应时间 | 解决时间 |
|-----|------|---------|---------|
| P0 | 核心功能不可用 | 15 分钟 | 2 小时 |
| P1 | 性能严重下降 | 1 小时 | 4 小时 |
| P2 | 单功能受限 | 4 小时 | 1 工作日 |
| P3 | 优化建议 | 2 工作日 | 1 周 |

## 备份策略

### 数据库备份
- 每日全量备份（凌晨 2 点）
- 每小时增量备份
- 备份保留 30 天

### 配置备份
- 实时同步配置文件
- Redis 每日 RDB 备份
- 配置版本控制
